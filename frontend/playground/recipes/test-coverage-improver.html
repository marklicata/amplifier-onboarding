Test Coverage Improver

Go from 60% to 90% test coverage without writing tests yourself.

What It Does

Analyzes your existing test suite to identify coverage gaps, then automatically generates comprehensive test cases to fill those gaps. The test-coverage agent finds untested code paths, edge cases, and error conditions, while modular-builder implements the actual test code.

Unlike simple coverage tools that just show metrics, this recipe actively improves your tests by writing new ones that are contextually appropriate and follow your existing test patterns.

How It Works

  1. Analyze Current Test Suite (test-coverage)

    Examines existing tests to understand patterns, framework usage, and coverage baseline.

  2. Identify Untested Code Paths

    Maps source code to test coverage, highlighting functions, branches, and conditions without tests.

  3. Assess Edge Cases & Error Conditions

    Identifies boundary conditions, error paths, and exceptional cases that need testing.

  4. Generate Test Specifications

    Creates detailed test case descriptions with inputs, expected outputs, and assertions.

  5. Implement Tests (modular-builder)

    Writes actual test code following your project's conventions and framework.

  6. Validate Tests Run Successfully

    Ensures generated tests pass and actually increase coverage as expected.

When to Use

  • Increase coverage for CI/CD requirements - Meet minimum coverage thresholds
  • Identify untested code paths - Find blind spots in your test suite
  • Generate edge case tests - Cover boundary conditions automatically
  • Create integration test suites - Build comprehensive integration tests
  • Validate error handling coverage - Ensure error paths are tested

Required Inputs

source_files Required

Source code files to analyze for test coverage.

Example: src/**/*.py

test_files Required

Existing test files to analyze.

Example: tests/**/*.py

framework Optional

Testing framework to use for generated tests.

Options: pytest, jest, junit, mocha

Default: pytest

target_coverage Optional

Target coverage percentage to achieve.

Default: 90
