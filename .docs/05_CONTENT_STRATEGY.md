## Voice & Tone

### Brand Voice Attributes

**Empowering, not Patronizing**
- ‚úÖ "You can build this"
- ‚ùå "Let us handle it for you"

**Visionary, not Buzzword-y**
- ‚úÖ "Imagine a world where roles dissolve"
- ‚ùå "Leveraging AI-powered synergies"

**Technical, not Academic**
- ‚úÖ "The bundle composes via deep merge"
- ‚ùå "Utilizing compositional paradigms"

**Pragmatic, not Dogmatic**
- ‚úÖ "This works for most cases"
- ‚ùå "You must always do X"

**Friendly, not Casual**
- ‚úÖ "Let's build something amazing"
- ‚ùå "Yo, this is gonna be lit"

### Writing Guidelines

**Do:**
- Use active voice
- Lead with value
- Show examples
- Explain tradeoffs
- Admit limitations
- Credit community

**Don't:**
- Use jargon without explanation
- Assume prior knowledge
- Hide complexity forever
- Overpromise capabilities
- Dismiss alternatives
- Take credit for user work

### Accessibility

**All content must:**
- Include alt text for images
- Provide captions for videos
- Use semantic HTML
- Support screen readers
- Offer text alternatives
- Use clear, simple language

## Crisis Communication

### Issues to Plan For

**Technical:**
- Service outage
- Security vulnerability
- Data loss
- Breaking changes

**Community:**
- Harassment incident
- Controversy over content
- Negative viral post

**Business:**
- Pricing changes
- Feature deprecation
- Competitive threat

### Response Framework

**Step 1: Acknowledge (Within 1 hour)**
- We're aware
- We're investigating
- We'll update soon

**Step 2: Update (Every 2-4 hours)**
- What we know
- What we're doing
- Next update time

**Step 3: Resolution (ASAP)**
- What happened
- What we fixed
- How we'll prevent it
- Apology if appropriate

**Step 4: Post-Mortem (Within 1 week)**
- Detailed timeline
- Root cause analysis
- Prevention measures
- Lessons learned
