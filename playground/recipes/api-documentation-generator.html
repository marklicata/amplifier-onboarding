API Documentation Generator

Ship beautiful API docs without writing a single sentence.

What It Does

Analyzes your API code and automatically generates comprehensive documentation including OpenAPI/Swagger specs, usage examples, error handling guides, and quick start instructions.

The explorer agent maps your API endpoints, parameters, and response types, while the zen-architect generates clear descriptions, use cases, and example requests that make your API easy to understand and use.

How It Works

  1. Parse API Endpoints (explorer)

    Scans code to identify all API routes, HTTP methods, and handler functions.

  2. Extract Parameters & Types

    Identifies request parameters, query strings, body schemas, and response types.

  3. Generate Descriptions (zen-architect)

    Creates clear, concise descriptions for each endpoint based on code analysis.

  4. Create Usage Examples

    Generates realistic request/response examples with different scenarios.

  5. Format & Validate

    Outputs OpenAPI/Swagger spec or Markdown, validates against standards.

When to Use

  • Generate OpenAPI/Swagger specs - Automatic API specification generation
  • Create API reference documentation - Complete endpoint reference with examples
  • Build getting started guides - Quick start tutorials for API consumers
  • Generate code examples - Request/response examples in multiple languages
  • Document REST endpoints - Comprehensive REST API documentation

Required Inputs

api_files Required

API code files to analyze and document.

Example: src/api/**/*.py

format Optional

Output documentation format.

Options: openapi, markdown, html

Default: openapi
