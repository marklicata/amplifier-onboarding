Security Audit Suite

Enterprise-grade security audit in minutes, not days.

What It Does

Performs a comprehensive security analysis of your codebase, covering common vulnerabilities, authentication/authorization patterns, dependency security, and configuration issues. This recipe orchestrates three specialized agents to provide enterprise-level security insights.

Results are prioritized by CVSS score and include exploitation scenarios, remediation guidance, and compliance checklists for OWASP Top 10 and other security frameworks.

How It Works

  1. Vulnerability Scanning (security-guardian)

    Scans for common vulnerabilities: SQL injection, XSS, CSRF, authentication flaws, etc.

  2. Authentication/Authorization Review (zen-architect)

    Analyzes auth patterns, session management, and access control logic for weaknesses.

  3. Dependency Security Check (security-guardian)

    Reviews package dependencies for known vulnerabilities (CVEs) and outdated versions.

  4. Configuration Review

    Checks for insecure configurations, exposed secrets, and misconfigurations.

  5. Compliance Checklist

    Generates OWASP Top 10 compliance report with pass/fail status.

  6. Prioritize & Report

    Sorts findings by CVSS score, provides exploitation scenarios and remediation steps.

When to Use

  • Pre-deployment security checks - Catch vulnerabilities before production
  • OWASP compliance validation - Ensure adherence to security standards
  • Dependency vulnerability scanning - Identify risky third-party packages
  • Authentication/authorization review - Validate access control implementation
  • Security best practices audit - Regular security health checks

Required Inputs

source_code Required

Source code directory to audit.

Example: src/

config_files Optional

Configuration files to check for security issues.

Example: config/*.yaml

compliance Optional

Compliance framework to validate against.

Options: owasp-top-10, pci-dss, hipaa, general

Default: owasp-top-10
