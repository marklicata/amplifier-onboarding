Bug Finder & Fixer

Find and fix bugs systematically with AI-powered debugging.

What It Does

Uses systematic, hypothesis-driven debugging to find bugs in your code. The bug-hunter agent analyzes error messages, code structure, and test failures to isolate root causes, then modular-builder generates and validates fixes.

This recipe doesn't just find obvious bugs - it discovers edge cases, race conditions, and subtle logic errors that are easy to miss in manual review.

How It Works

  1. Reproduce the Issue (bug-hunter)

    Analyzes error messages and stack traces to understand the symptom.

  2. Hypothesis Generation

    Formulates hypotheses about potential root causes based on code analysis.

  3. Isolate the Cause

    Tests each hypothesis systematically to identify the actual bug.

  4. Generate Fix (modular-builder)

    Creates code fixes that address the root cause, not just symptoms.

  5. Validate the Fix

    Ensures fix resolves issue without introducing new bugs.

  6. Add Regression Tests

    Creates tests to prevent this bug from reoccurring.

When to Use

  • Debug failing tests - Understand why tests are breaking
  • Find edge case bugs - Discover bugs in unusual conditions
  • Analyze error logs - Investigate production errors systematically
  • Fix runtime errors - Resolve crashes and exceptions
  • Systematic debugging workflow - Methodical bug hunting process

Required Inputs

code_file Required

Code file containing the bug.

Example: src/app.py

error_message Optional

Error message or symptom description.

Example: KeyError: 'user_id'

test_file Optional

Test file showing the failure (helps with reproduction).

Example: tests/test_app.py
