Accessibility Statement

Our Commitment

Amplifier is committed to ensuring digital accessibility for people with disabilities. We are continually improving the user experience for everyone and applying the relevant accessibility standards.

Conformance Status

The Web Content Accessibility Guidelines (WCAG) defines requirements for designers and developers to improve accessibility for people with disabilities. It defines three levels of conformance: Level A, Level AA, and Level AAA.

Amplifier Onboarding is designed to be conformant with WCAG 2.1 Level AA.

Accessibility Features

  • Keyboard Navigation - All functionality available via keyboard
  • Screen Reader Support - Semantic HTML and ARIA labels throughout
  • High Contrast Mode - Enhanced borders and contrast in high contrast mode
  • Reduced Motion - Respects prefers-reduced-motion preference
  • Focus Indicators - Visible focus states for keyboard navigation
  • Responsive Text - Text scales with user preferences
  • Skip Links - Skip to main content on every page
  • Color Independence - Information not conveyed by color alone

Known Limitations

Despite our best efforts, some limitations may be present:

  • Interactive recipe execution (Phase 1) - Will be tested for accessibility when implemented
  • Visual recipe builder (Phase 3) - Will have keyboard-accessible alternative
  • Some third-party embedded content may have accessibility limitations

Feedback

We welcome your feedback on the accessibility of Amplifier. Please let us know if you encounter accessibility barriers:

  • GitHub Issues: Report accessibility issues
  • Email: accessibility@microsoft.com

We try to respond to feedback within 2 business days.

Technical Specifications

Accessibility of Amplifier relies on the following technologies:

  • HTML5
  • WAI-ARIA
  • CSS3
  • JavaScript (progressive enhancement)

These technologies are relied upon for conformance with the accessibility standards used.

Assessment Approach

Microsoft assessed the accessibility of Amplifier by the following approaches:

  • Self-evaluation
  • Automated testing tools (Lighthouse, axe-core)
  • Manual keyboard testing
  • Screen reader testing (NVDA, VoiceOver)
